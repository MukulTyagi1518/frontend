# Newsletter Management

## Subscribe to Newsletter

<ParamField body="email" type="string" required>
  The email address to subscribe to the newsletter
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the subscription was successful
</ResponseField>

<ResponseField name="message" type="string">
  A message describing the result of the operation
</ResponseField>

<RequestExample>
```bash Example Request
curl --request POST \
  --url 'https://api.example.com/api/fitnearn/web/newsletter-subscribe' \
  --header 'Content-Type: application/json' \
  --data '{
    "email": "user@example.com"
  }'
```
</RequestExample>

<ResponseExample>
```json Response
{
  "success": true,
  "message": "Successfully subscribed to newsletter"
}
```
</ResponseExample>

## Error Handling

All endpoints follow standard HTTP status codes:

| Status Code | Description                                                   |
|------------|---------------------------------------------------------------|
| 200        | Operation completed successfully                              |
| 400        | Bad request - Invalid parameters                              |
| 401        | Unauthorized - Authentication required                        |
| 404        | Resource not found                                           |
| 500        | Internal server error                                        |

## Notes

- Email addresses must be valid and properly formatted
- Verification tokens are valid for 24 hours
- Newsletter broadcasts are rate-limited
- Unsubscribe links are automatically included in all newsletters
